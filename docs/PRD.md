# HabitFlow (habitflow)

# PRD
## Background / Problem
- 목표로 제시된 사용 행동 기준은 **“하루 평균 30초 내로 습관 체크 + 진행 상황 확인”**이며, 제품 성공 지표는 **출시 3개월 내 DAU 3,000명 / 30일 리텐션 40%**입니다.
- 기존 습관 앱(Habitica, Streaks 등)은 기능이 많고 초기 설정이 번거로워 20–30대 직장인/대학생이 빠르게 이탈합니다. 사용자는 “매일, 바로, 한 번 탭으로” 체크하고 흐름을 끊지 않고 성취(스트릭/달성률)를 확인하길 원합니다. 또한 지하철/강의실 등 **오프라인 환경에서도 기록이 보장**되어야 합니다.

## Goal (1-sentence product definition)
출시 3개월 내 **DAU 3,000명**, **30일 리텐션 40%**를 달성하고, 사용자가 **하루 평균 30초 이내**에 오늘 습관 체크와 진행 확인을 끝낼 수 있는 최소 기능 습관 추적 앱을 제공한다.

## Non-goals
- 친구/커뮤니티 등 **소셜 기능**
- 레벨업/배지/캐릭터 등 **복잡한 게임화**
- 습관별 **메모/저널**(서술형 기록)
- **웹 버전** 제공
- 타이머/스톱워치
- 카테고리/태그 시스템
- (모바일 MVP) **푸시 알림** 및 위젯

## Target Users (personas + use cases)
- **지수(27, IT 기업 주니어 개발자)**  
  - 상황/역할: 출퇴근 지하철에서 짧게 루틴 관리  
  - 페인포인트: 기존 앱은 설정/화면이 복잡해 3일 내 포기  
  - 사용 사례: 오늘 해야 할 습관을 앱 열자마자 보고 **한 번 탭으로 체크** + 스트릭 확인
- **민서(24, 대학생)**  
  - 상황/역할: 강의실/도서관 등 데이터가 불안정한 환경에서 루틴 관리  
  - 페인포인트: 인터넷이 없으면 기록이 누락되거나 동기화가 깨짐  
  - 사용 사례: **오프라인 체크 후 자동 동기화**, 주간 달성률로 패턴 확인

## Target Market
- 지역: **대한민국(South Korea)**
- 초기 타겟: **20–30대 직장인/대학생**, “복잡한 습관 앱에 지친 사용자”
- 핵심 가치 제안: **3초 체크(탭 1번) + 스트릭/주간 시각화 + 오프라인 기록 보장**

## Data Entities (nouns with key fields)
- **Habit**
  - `id: string`
  - `title: string`
  - `createdAt: string (ISO)`
  - `isActive: boolean`
  - `currentStreak: number`
  - `bestStreak: number`
- **CheckIn**
  - `id: string`
  - `habitId: string`
  - `date: string (YYYY-MM-DD, local)`
  - `checkedAt: string (ISO)`
  - `isSynced: boolean` (오프라인 큐/동기화 상태)
- **Subscription**
  - `tier: 'free' | 'premium'`
  - `expiresAt: string | null (ISO)`
- **HabitRecommendation**
  - `id: string`
  - `title: string`
  - `isPopular: boolean`
  - `sortOrder: number`

## Core Flow (numbered steps)
1. **첫 실행(OnboardingModal)**: 인기 습관 3–5개 추천 리스트 표시 + “직접 만들기”
2. 사용자가 **2개 선택/생성** 후 완료 → **Today 탭(오늘의 체크리스트)**로 진입
3. Today에서 습관 항목을 **한 번 탭** → 해당 날짜 CheckIn 생성/토글, **체크 완료 피드백** 및 스트릭 +1 반영
4. 하루 습관을 모두 체크하면 **완료 메시지** 표시
5. 사용자가 **Stats 탭**으로 이동 → **주간(월~일) 달성률 막대 그래프** 확인 (무료 제공 범위)
6. 사용자가 더 많은 습관 추가/월별 통계/패턴 분석을 보려 하면 **Profile 탭에서 프리미엄 업그레이드** 진입 및 구독 상태 반영

## Success Metrics (measurable)
- **DAU 3,000명** (출시 3개월 내)
- **30일 리텐션 40%** (출시 3개월 내)
- **평균 세션 길이 < 1분** (체크 중심)
- **무료→유료 전환율 5%**
- **주간 체크율 70% 이상** (사용자당 주 5일 이상 체크)

## MVP Scope (exhaustive feature list)
- **오늘의 체크리스트(Today)**: 앱 실행 시 오늘 습관 목록 즉시 노출, **탭 1회로 체크/해제**
- **스트릭 카운터**: 습관별 `currentStreak` 및 `bestStreak(Personal Best)` 표시/갱신
- **주간 달성률 그래프(Stats)**: 월~일 막대 그래프로 주간 달성률 시각화(무료 범위)
- **오프라인 체크 + 자동 동기화**: 오프라인에서도 CheckIn 저장(`isSynced=false`) 후 온라인 복귀 시 자동 재시도
- **추천 습관 온보딩**: 첫 실행 시 추천 3–5개 제안 + 빠른 시작(빈 화면 방지)
- **프리미엄 구독(Freemium 게이팅)**: 무료(습관 3개 제한/주간 통계) vs 프리미엄(무제한 습관/월별 통계/습관별 패턴 분석 화면 접근)

## Target Audience & Marketing
- 타겟: 20–30대 직장인/대학생, 기존 습관 앱의 복잡함에 피로한 사용자
- 메시지: **“3초면 충분해”**, “탭 한 번으로 끝”
- 톤앤매너: minimal
- 채널: 카카오톡, 네이버 블로그, 유튜브(국내)

## Monetization Strategy
- **freemium**: 무료는 습관 3개 + 주간 통계 제공, 프리미엄은 무제한 습관 + 월별 통계 + 습관별 달성 패턴 분석 제공  
- 결제는 **App Store / Google Play 인앱 구독**을 기본으로 한다(플랫폼 수수료 15–30%).  
- (참고) brief의 결제 제공사(toss/kakaopay/stripe) 사용 여부는 정책/구현 방식 재검토가 필요.

## Assumptions
1. 사용자는 하루에 최소 1회 앱을 열고 Today에서 체크하는 행동이 핵심이다.
2. 체크 데이터는 “당일 1회 완료” 단위로 단순화해도 MVP 가치가 유지된다.
3. 오프라인에서 생성된 CheckIn은 온라인 복귀 시 서버에 순차 동기화하면 충분하다.
4. 동기화 충돌은 “동일 날짜/습관은 마지막 체크 상태를 우선” 규칙으로 MVP에서 처리 가능하다.
5. 무료 사용자의 습관 3개 제한은 전환에 의미 있는 압박을 제공한다.
6. 주간 통계만으로도 무료 사용자의 초기 유지(리텐션)에 도움이 된다.
7. 추천 습관 3–5개는 고정된 인기 리스트로 시작해도 된다(개인화는 이후).
8. iOS/Android 모두 동일 UX(탭, 스와이프)로도 사용성 목표(30초 내 완료)를 달성할 수 있다.
